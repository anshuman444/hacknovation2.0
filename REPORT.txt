DEFIGUARDIAN: TECHNICAL PROJECT REPORT (HACKATHON UNICORN EDITION)
================================================================

1. PROJECT OVERVIEW
-------------------
DefiGuardian is an AI-powered smart contract risk intelligence platform designed to bridge the gap between abstract code audits and real-world historical threats. It transforms standard security scans into a Forensic Intelligence Engine that provides comparative logic against historical hacks.

2. HOW IT WORKS (THE PIPELINE)
------------------------------
The platform operates as a secure, automated pipeline for smart contract auditing and distribution:

STEP 1: INGESTION & HEURISTICS
- User uploads/pastes Solidity smart contract source code.
- The system performs a 'Local Heuristic Scan' to identify immediate critical patterns (Reentrancy, Self-destruct, Integer Overflows).

STEP 2: FORENSIC AI ANALYSIS
- The code is sent to the 'Advanced AI' engine.
- The AI performs 'Deep Packet Inspection' to match vulnerabilities to a historical DNA database (The DAO, Mango Markets, etc.).
- It generates 'Forensic Patterns' and 'Similarity Rationales' explaining WHY the code is vulnerable.

STEP 3: DECENTRALIZED VALIDATION (EYE ON UNITY)
- Results are cryptographically signed and sent for validation via EigenLayer and Cartesi.

STEP 4: MARKETPLACE PUBLICATION
- Once validated, the contract is published to the Marketplace with a full Security Manifest.

3. UNICORN FEATURES (FORENSIC INTELLIGENCE)
--------------------------------------------
A. VISUAL INTELLIGENCE DASHBOARD:
- Security Posture Matrix: A high-impact grid of technical security markers (Non-Custodial, Logic-Gated, State-Verified).
- Threat Identity Matrix: Visual density charts mapping vulnerabilities against historical hack archetypes.
- Executive Forensic Summary: A "Judge-Ready" strategic brief at the top of every audit report.

B. COMPARATIVE FORENSICS:
- Exploit DNA Matching: Direct technical links to famous hacks (The DAO, Poly Network, etc.).
- Threat Velocity Gauges: Animated indicators predicting the speed and scale of potential fund drains.

4. COMPONENT BREAKDOWN
-----------------------
CLIENT-SIDE:
- `navbar.tsx`: The primary navigation hub; features high-impact cyberpunk animations and branding alerts.
- `home.tsx`: The launchpad; handles contract ingestion and initiates the "One-Click" pipeline.
- `results.tsx`: The Forensic Visualizer; displays the 'Judge's Brief' summary, historical DNA matches, and animated 'Threat Velocity' gauges.
- `marketplace.tsx`: The Trust Registry; displays all verified contracts with scrollable Forensic Manifests for potential buyers/users.

SERVER-SIDE:
- `routes.ts`: The Orchestrator; handles AI communication, manages the "Validate-to-Publish" state machine, and implements the Forensic fallback logic.
- `storage.ts`: The Memory Vault; manages the real-time state of users, audits, and the verified contract registry.
- `schema.ts`: The Blueprint; defines the JSON-B structures that store forensic data (Similarity Rationales, Impact Scores).

8. INFRASTRUCTURE & DATA INTEGRITY (THE ENGINE ROOM)
---------------------------------------------------
A. ZOD SCHEMA (THE GUARDIAN):
- Role: Every piece of forensic data (Similarity Rationales, Impact Projections) is strictly validated via Zod Schemas before it touches the frontend.
- Benefit: Prevents "AI Hallucinations" from corrupting the UI; ensures judges see zero-error, mathematically structured security manifests.

B. DRIZZLE ORM & STORAGE (THE MEMORY VAULT):
- Role: A high-performance SQL layer that stores all audit history and Marketplace entities.
- Benefit: Provides a "Permanent Forensic Record" of every scan, making the security data queryable and ready for institutional-grade compliance.
